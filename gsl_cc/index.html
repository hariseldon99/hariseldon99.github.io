Introduction

This page instructs viewers to cross-compile Windows DLL's for the GNU Scientific Library on a Linux system.

Requirements

You will need the following resources to perform the cross-compilation.

  1. A running instance of a Linux operating system with a shell like bash. Any Linux distribution should do
  2. The Git distributed version control system
  3. The M cross environment (MXE)
  4. The source code for the GNU Scientific Library
  5. Access to the target Windows operating system

Getting Started #back to top

Install git on Linux MXE Tutorial

On a Linux operating system, install git and use it to clone the MXE repository and install it as instructed on their website. Assuming that you have downloaded MXE to the directory $MXE_DIR (set the shell variable MXE_DIR, if necessary). To install MXE, start a bash shell and run 

1
2
 
$ git clone https://github.com/mxe/mxe.git
$ MXE_DIR=$PWD/mxe

Compile the Basic MXE Toolset #back to top

Assuming that you have downloaded MXE to the directory $MXE_DIR, you can compile MXE as follows.

If you want to compile 32-bit libraries, install the most basic tools in MXE with the command 
1
2
 
$ cd $MXE_DIR
$ make cc

If you want to compile 64-bit libraries, install the most basic tools in MXE with the command 
1
2
 
$ cd $MXE_DIR
$ make MXE_TARGETS=x86_64-w64-mingw32.static cc

After running the "make", append the PATH variable, as they instruct, to 
$ PATH=$MXE_DIR/usr/bin:$PATH/

Download and Unpack GSL Source Code #back to top

Download and Unpack the GSL Source Code

Download GSL Unpacking tar.gz archives

Download and unpack the GSL source code from here. Note that, as of the writing of this page, the latest version of GSL is gsl-2.5, and the file to download is "gsl-2.5.tar.gz". Please update your commands to the latest working version. Unpack the source code with the command 

1
 
$ tar xvzf gsl-2.5.tar.gz -C $MXE_DIR

Compile and Install GSL Libraries for Windows #back to top

Navigate your bash shell to the root directory of the unpacked source and run the following command

For 32-bit compilation, run 
1
2
3
4
 
$ cd $MXE_DIR/gsl-2.5/
$ ./configure --host=i686-w64-mingw32.static --enable-shared --disable-static --prefix=$MXE_DIR/gsl-2.5_i686/
$ make
$ make install

For 64-bit compilation, run 
1
2
3
4
 
$ cd $MXE_DIR/gsl-2.5/
$ ./configure --host=x86_64-w64-mingw32.static --enable-shared --disable-static --prefix=$MXE_DIR/gsl-2.5_x86_64/
$ make
$ make install

This will cross-compile the windows dll libraries and install them, together with the headers, to "$MXE_DIR/gsl-2.5_i686" or "$MXE_DIR/gsl-2.5_x86_64", depending on whether you're compiling for 32-bit (i686) or 64-bit (x86_64).

Install the GSL Libraries on Windows #back to top

Copy the directories therein to your windows operating system and put them anywhere. I'm assuming that they have been put in "C:\Program Files\gsl-2.5\". You may adjust this path to your install location.

Finally, make sure to append your PATH environment variable on windows to point to the bin and lib directories, otherwise the compiled binaries won't run. The instructions to access the environment variable settings are different for different versions of windows. Goto this link and follow instructions for your operating system. Alternatively, see the different instructions for doing this in different versions of windows. Note that you will need administrator access to do this.Once you have accessed the settings for "PATH", add to the existing variable the directories as shown in the screenshots below.

How to set the PATH variable on windows

Windows 8 and Windows 10 - #top

  • Step 1 - In Search, search for and then select: System (Control Panel)
  • Step 2 - Click on "Advanced system settings"
  • Step 3 - Click Environment Variables. In the section System Variables, find the PATH environment variable and select it. Click Edit. If the PATH environment variable does not exist, click New.
  • Step 4 - In the Edit System Variable (or New System Variable) window, specify the value of the PATH environment variable. In this case, you have to add two paths to the existing PATH value as shown in the screenshot above (adjust the value for your installation). Click OK. Close all remaining windows by clicking OK.
  • Step 5 - In some installations, there may not be a dialog screen for setting the PATH variable. In that case, go to the "Environment Variables" window as in step 3, highlight the "Path" variable in the "System variables" section and click the Edit button. Add or modify the path lines with the paths to gsl bin and lib directories. Note that each folder path is separated by a semicolon. So, for this example, simply update the existing variable with "C:\Program Files\gsl-2.5\bin;C:\Program Files\gsl-2.5\lib" (without the quotes).

Windows Vista and Windows 7 #top

  • Step 1 - From the Desktop, right-click the Computer icon and select Properties. If you don't have a Computer icon on your desktop, click the Start button, right-click the Computer option in the Start Menu, and select Properties.
  • Step 2 - Click the Advanced System Settings link in the left column.
  • Step 3 - In the System Properties window, click on the Advanced tab, then click the Environment Variables button near the bottom of that tab.
  • Step 4 - In the Environment Variables window (pictured above), highlight the Path variable in the "System variables" section and click the Edit button. Add or modify the path lines with the paths to the gsl bin and lib directories. Each different directory is separated with a semicolon. So, for this example, simply update the existing variable with "C:\Program Files\gsl-2.5\bin;C:\Program Files\gsl-2.5\lib" (without the quotes).

Windows Command Line / MS DOS #top

  • Download the SetEnv Program
  • Step 1 - Download the SetEnv program (an EXE file) from this website:
  • Step 2 - Run the SetEnv EXE from the command line thus: 
    SetEnv -a PATH %"C:\Program Files\gsl-2.5\bin;C:\Program Files\gsl-2.5\lib"
  • Adjust the value(s) for your installation.

Build and Run C Programs with GSL #back to top

Note that, if you have installed the gsl folder in "C:\Program Files\gsl-2.5\", then

  1. The header files are installed in "C:\Program Files\gsl-2.5\include\"
  2. The dynamically linked library archives are installed in "C:\Program Files\gsl-2.5\lib\"
  3. The running binaries (gsl-config, gsl-histogram etc.) and dll libraries are in "C:\Program Files\gsl-2.5\bin\"

When configuring your IDE, use these values, adjusted for your installation. A nice IDE for Windows is Code::Blocks. Instructions to downloading, installing and using it are given below.

  • Code::Blocks Webpage Installing a supported compiler Using libraries with Code::Blocks
